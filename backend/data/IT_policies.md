# IT & Security Policy Document

## 1. Device & Asset Usage Policy

- Company-issued devices are for official work only.
- Personal software installation is not allowed without IT approval.
- Devices remain company property and can be audited at any time.

---

## 2. Software Access Control

- Access to software is role-based (RBAC).
- Developers receive access only to required environments.
- Admin privileges are strictly restricted to IT team.

### 2.1 Unauthorized Tools
- Use of unauthorized cloud services (e.g., personal drives for work data) is prohibited.
- External AI tools must not be used with sensitive company data.

---

## 3. Password & Authentication Policy

- Minimum 8 characters with complexity requirements.
- Password expiration: 90 days.
- Reuse of last 5 passwords is blocked.
- Multi-factor authentication (MFA) is mandatory for all systems.

---

## 4. Network & VPN Policy

- VPN must be active when accessing internal systems remotely.
- Public Wi-Fi usage requires VPN at all times.
- Sharing VPN credentials is a critical security violation.

---

## 5. Data Security Policy

### 5.1 Data Classification
- Public: Safe to share externally
- Internal: Company-only access
- Confidential: Restricted access (HR, finance, projects)

### 5.2 Data Handling Rules
- Confidential data must not be stored on personal devices.
- Screenshots of sensitive dashboards are prohibited.
- Data leaks will result in disciplinary/legal action.

---

## 6. System Access & Logging

- All system actions are logged and monitored.
- Access to production systems requires approval.
- Any suspicious activity is reviewed by IT security team.

---

## 7. Incident Reporting

- Security incidents must be reported within 1 hour.
- Includes: data loss, unauthorized access, malware detection.
